# Zomboid_Migration
Project Zomboid Migration mod

ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ

Разработка модификации "Realistic Forest Migration" для Project Zomboid (Build 41)

1. ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ)

1.1. Введение и актуальность исследования
В текущей реализации архитектуры игры Project Zomboid (Build 41) подсистема
мета-популяции зомби использует математическую модель равномерного перераспределения
виртуальных сущностей между макро-ячейками (cells) карты. Данный алгоритм направлен на
поддержание постоянного игрового напряжения (пассивного прессинга игрока) посредством
заполнения зачищенных зон мигрирующими группами из соседних закрытых областей. Однако при
установке экстремальных множителей популяции (режим «Песочницы» с множителем X = 16.0 )
стандартный алгоритм демонстрирует логическое противоречие: зачищенные игроком
труднопроходимые лесные массивы за короткий промежуток времени заполняются тысячами
автономных сущностей из смежных секторов. Это полностью нивелирует стратегическую ценность
выживания вне городских агломераций, ломает игровую логику реализма и вызывает избыточную
нагрузку на сборщик мусора Java VM из-за непрерывного спавна/деспавна сотен 3D-моделей
персонажей.

Цель работы: Проектирование и программная реализация мода на языке
Lua, модифицирующего логику мета-миграции игровых сущностей для обеспечения возможности
долговременного удержания и очистки изолированных и лесных зон без нарушения общего
баланса симулятора.

1.2. Функциональные требования к программному модулю
Динамическая фильтрация по лесистости (Forest Density Filter): Модуль должен
выполнять потоковый анализ структуры тайлов целевого чанка. Если коэффициент плотности
деревьев превышает заданный порог, входящий поток миграции должен снижаться по
экспоненциальной зависимости.

Эффект «Памяти очистки» (Zone Security Buffer): Создание временного триггерного буфера
в радиусе активности игрока. Ячейки, в которых зафиксировано массовое уничтожение зомби
игроком, должны блокировать входящую миграцию на фиксированный период времени для
симуляции реального истощения локального пула.

Консервация пула (No In-Air Spawning): Запрет скрытой генерации зомби внутри закрытых
ячеек, не имеющих физической связи с донорскими городскими зонами. Миграция должна
строго интерпретироваться движком как пространственное перемещение существующих
виртуальных объектов, а не как отложенный респавн.

Оптимизация высоконагруженных вычислений: Алгоритм проверки не должен вызывать
каскадных задержек (микрофризов) главного потока при итерации по массивам объектов на
популяции х16. Применяется событийная модель обработки вместо безусловного
ежесекундного тика.

1.3. Архитектура и ограничения интеграции
В ходе исследования исходного кода игры выявлено, что подсистема популяции имеет гибридную
архитектуру. Вычислительное ядро скомпилировано в нативные библиотеки (C/C++), а
интерфейсный слой реализован на Java. Интеграция Lua-мода будет осуществляться
посредством перехвата функций Java-слоя, выступающих мостом (JNI Wrapper) для нативного
кода. Прямая модификация бинарных файлов PZPopMan64.dll/.so исключается в целях
сохранения кроссплатформенности и совместимости со Steam Workshop.

4. МАТЕМАТИЧЕСКАЯ И СТРУКТУРНАЯ МОДЕЛЬ РЕАЛИЗАЦИИ
Предлагается следующая расчетная модель
функционирования фильтра. Допустим, стандартный шаг миграции пытается перевести объем
зомби Vinit в выбранную ячейку. Мы вводим коэффициент лесистости ячейки Kf , который

рассчитывается как:

Kf = Ntree / Ntotal
Где Ntree — количество тайлов деревьев и густой растительности в чанке, а Ntotal — общее
количество тайлов в чанке ( 10 × 10 = 100 ). Итоговый объем спавна мигрирующих зомби в лесу
Vfinal будет определяться уравнением:

Vfinal = Vinit × (1 − Kf)α
Где α — коэффициент жесткости фильтрации (рекомендуемое значение для популяции х16: α =
3.0 ). Таким образом, при плотности леса в 80% ( Kf = 0.8 ), входящий поток зомби будет снижен
более чем на 99%, что обеспечит полную изоляцию лесного лагеря игрока и гарантирует
выживаемость системы.
